# Verification Tiers (für Korrekturen in Gruppen)

Nutze Tiers, um zu entscheiden, wann du andere in Gruppen **sicher** korrigieren kannst.

- **v0**: Memory / Vermutung → nie als Basis für Korrekturen
- **v1**: Chat-Zitat / Screenshot → nur auf Anfrage oder als Zusammenfassung
- **v2**: Config/State Read → ok
- **v3**: Logs/Status Endpoint → ok

Daumenregel:

- Unaufgeforderte Korrekturen in Gruppen: nur **v2/v3**.
